# GIS Tools Overview 🛠️

Geospatial architectures utilize specialized servers, APIs, and portals to publish spatial databases and file layers as web services. This section compiles learning resources for the leading open-source packages in the ecosystem.

---

## 🧭 Servicers & Portals

Select a tool to explore its configuration and learning materials:

*   **[GeoServer](geoserver/workspaces.md)**: The industry-standard Java web application to publish, edit, and style (SLD) spatial layers as legacy WMS/WFS protocols.
*   **[QGIS](qgis/interface.md)**: The leading open-source desktop GIS software to load, style, visual debug, and analyze spatial vector and raster data.
*   **[TiTiler](titiler.md)**: A modern, high-performance dynamic tile server built on FastAPI and Rasterio, optimized for serving Cloud-Optimized GeoTIFFs (COG).
*   **[pgTileserv](pgtileserv.md)**: A lightweight Go microservice that queries PostGIS tables and dynamically serves Mapbox Vector Tiles (MVT) without complex setups.
*   **[pgFeatureserv](pgfeatureserv.md)**: A Go microservice designed to serve PostGIS data tables as modern OGC Features API endpoints (JSON).
*   **[OGC APIs](ogc-apis/basics.md)**: Learn about the new generation of open standards replacing WMS/WFS with clean web-friendly REST APIs.
*   **[pygeoapi](pygeoapi/setup.md)**: A Python server execution of OGC API standards.
*   **[GeoNode](geonode/setup.md)**: A complete Django-based geospatial content management platform to host, search, and manage portal layers and maps.
*   **[MapStore](mapstore/setup.md)**: A modular web mapping client to build interactive maps, dashboards, and spatial widgets.
